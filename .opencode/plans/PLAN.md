# План рефакторинга konkurent

## Цель

Избавиться от жёстко зашитой логики построения Excel-таблицы, заменив её
JSON-конфигом, и добавить три GUI-фичи: имя заявки, редактор позиций заявки,
скрытую строку с именем файла.

## Архитектура после рефакторинга

```
gui.py (CustomTkinter)
  ├── вкладка «Позиции заявки»   (редактор B/C)
  ├── вкладка «Выбор папки»      (скан PDF + имя поставщика)
  └── вкладка «Drag && Drop»     (DND + имя поставщика)
       │
       ▼
recog.py
  ├── process_pdf_file()     — извлечение PDF → (df, orig, bez_nds)
  ├── fill_template()        — создаёт книгу из config.json + данные
  └── build_workbook()       — [NEW] строит голую книгу по конфигу
       │
       ▼
config.json — вся структура таблицы (колонки, строки, мета-секция, тотал)
template.xlsx — удаляется (больше не нужен)
```

## Сводка шагов

| Шаг | Ветка | Описание |
|-----|-------|----------|
| 1 | `feat/json-config` | JSON-конфиг + динамическое построение книги, удаление template.xlsx |
| 2 | `feat/request-name` | Поле имени заявки в GUI |
| 3 | `feat/request-items` | Третья вкладка — редактор позиций заявки |
| 4 | `feat/hidden-filename-row` | Скрытая строка с именем файла над row 2 |

## Процесс выполнения

Каждый шаг разбит на подшаги. Между подшагами — **точка ручного контроля**:

| Символ | Кто делает | Что происходит |
|--------|-----------|----------------|
| ▶ | opencode | opencode вносит изменения в код |
| ⏸ **Контроль** | opencode | opencode останавливается и ждёт вашей команды |
| ✅ **Проверка** | **ВЫ** | Запускаете GUI, проверяете результат |
| 🔀 **Мёрж** | **ВЫ** | Утверждаете мерж в main |

### Команды для управления

| Ваша команда | Что делает opencode |
|-------------|---------------------|
| «Продолжай» | Переходит к следующему подшагу |
| «Отмена, шаг N сломан» | Останавливается, ждёт инструкций по откату |
| «Утверждаю» | Выполняет push после мержа |
